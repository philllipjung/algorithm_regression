## 1. 알고리즘의 처리 과정 및 알고리즘 간 전달 값

### 1.1 전체 처리 과정 흐름

이 알고리즘은 다음과 같은 5단계 처리 과정을 거칩니다:

```
[데이터 로드 및 전처리] → [Random Forest 특성 중요도 분석] →
[Lasso 회귀 분석] → [Step Forward K-Fold 선택] → [상관관계 분석 및 결과 저장]
```

### 1.2 단계별 처리 과정 및 전달 값

### **단계 1: 데이터 로드 및 전처리 (라인 230-289)**

**처리 과정:**
- HDFS에서 입력 데이터를 로드
- 데이터 유효성 검사 (데이터 비어있음, 행 수 10개 미만, 표준편차 0 등)
- 결측값 처리 및 불필요한 컬럼 제거
- 표준화(StandardScaler) 적용

**전달 값:**
- `X_R2SM_si`: 정규화된 입력 특성 변수들 (예: 온도, 압력, 시간 등 공정 파라미터)
- `y_R2SM_si`: 정규화된 타겟 변수 (수율 또는 품질 지표)

### **단계 2: Random Forest 특성 중요도 분석 (라인 290-306)**

**처리 과정:**
- `bootstrap_rf` 함수를 10번 반복 실행 (Parallel 처리)
- 각 반복에서 데이터를 70:30으로 분할하여 Random Forest 훈련
- 특성 중요도(importance)와 표준편차(std) 계산
- 스케일링된 중요도(scaled importance) 계산: `importance / std`

**전달 값:**
- `stats_of_importance_each_variable`: 각 변수의 중요도 통계 (mean, std, min, max 등)
- `rf_feature_impt_mean_over_zero`: 평균 중요도가 0보다 큰 변수들만 필터링

**알고리즘별 중요 파라미터:**
```python
# bootstrap_rf 함수 파라미터
n_estimators=100          # 트리 개수
oob_score=True           # Out-of-Bag 점수 계산
n_jobs=-1                # 모든 CPU 코어 사용
warm_start=True          # 이전 훈련 재사용
test_size=0.3            # 테스트 데이터 30%
```

### **단계 3: Lasso 회귀 분석 (라인 307-346)**

**처리 과정:**
- RF에서 선별된 변수들로 Lasso 데이터셋 생성 (`create_lasso_dataset`)
- 100개의 alpha 값에 대해 Lasso 점수 계산 (Parallel 처리)
- 최적 alpha 값 선택 (테스트 점수가 최대가 되는 값)
- 최적 Lasso 모델 훈련 및 0이 아닌 계수 추출

**전달 값:**
- `lasso_features_1st`: Lasso에서 선택된 변수들과 계수 점수
- `XX_R2SM_si`: 정규화된 Lasso 입력 데이터

**과도한 훈련 방지 방법:**
- **alpha 파라미터 규제**: L1 규제를 통해 불필요한 변수의 계수를 0으로 축소
- **교차 검증**: 70:30 데이터 분할로 훈련/테스트 성능 모니터링
- **최적 alpha 선택**: 테스트 점수가 최대가 되는 alpha 선택
- **조기 종료**: max_iter=1000으로 반복 횟수 제한

```python
# Lasso 주요 파라미터
alpha=optimal_alpha       # 규제 강도 (자동 선택)
tol=0.0005               # 수렴 허용 오차
max_iter=1000            # 최대 반복 횟수
```

### **단계 4: Step Forward K-Fold 선택 (라인 348-371)**

**처리 과정:**
- Lasso 선택된 변수들로 데이터 재구성
- `step_forward_k_fold` 함수로 순방향 변수 선택 수행
- K-Fold 교차 검증으로 과적합 방지
- K-Fold Adjusted R-squared가 최대가 되는 모델 선택

**전달 값:**
- `total_result`: 각 단계별 모델 정보와 성능 지표
- `lasso_forward`: 최종 선택된 변수들과 계수

**과도한 훈련 방지 방법:**
- **K-Fold 교차 검증**: 데이터를 K개로 분할하여 교차 검증
- **Adjusted R-squared**: 변수 개수로 조정된 R-제곱 사용 (과적합 방지)
- **조기 종료 조건**: K-Fold 점수가 10번 연속 개선되지 않으면 종료
- **Tol 허용 오차**: 개선량이 tol=0.000 이하면 중지

```python

```

### **단계 5: 상관관계 분석 및 결과 저장 (라인 373-476)**

**처리 과정:**

- 중앙값 필터링과 평균 상관계수 계산
- 결과를 Hive 테이블에 저장

**최종 전달 값:**

## 2. 알고리즘별 중요 파라미터 및 과도한 훈련 방지 방법

### 2.1 Random Forest (bootstrap_rf)

**중요 파라미터:**
```python
n_estimators=100          # 생성할 트리 개수
oob_score=True           # OOB 점수로 일반화 성능 추정

```

**과도한 훈련 방지:**
1. **Bootstrap Sampling**: 각 트리가 다른 데이터 샘플로 훈련되어 다양성 확보
2. **OOB Score**: 훈련에 사용되지 않은 샘플로 검증
3. **10번 반복 실행**: 다양한 시드로 실행하여 중요도의 안정성 확인
4. **표준화 중요도**: `importance / std`로 신뢰도 조정
5. **병렬 처리로 다양한 초기화**: 서로 다른 random_state로 동시 실행

### 2.2 Lasso Regression

**중요 파라미터:**
```python
alpha=optimal_alpha      # L1 규제 강도 (교차 검증으로 선택)

max_iter=1000           # 최대 반복 횟수 제한

fit_intercept=False     # 데이터가 이미 정규화됨
```

**과도한 훈련 방지:**
1. **L1 규제 (Lasso)**: 불필요한 특성의 계수를 정확히 0으로 만들어 특성 선택 자동화
2. **Alpha 그리드 서치**: 100개의 alpha 값에 대해 교차 검증
3. **테스트 점수 기반 선택**: 훈련 점수가 아닌 테스트 점수로 최적 alpha 선택
4. **정규화된 데이터**: 모든 특성을 같은 스케일로 변환
5. **조기 종료**: tol=0.0005로 수렴 조건 설정

### 2.3 Step Forward K-Fold Selection

**중요 파라미터:**
```python
k=5                     # 5-Fold 교차 검증
tol=0.000              # 개선량 임계값 (허용 오차)
fit_intercept=False    # 정규화된 데이터이므로 절편 없음
n_jobs=-1              # 선형 회귀 병렬 처리
```

**과도한 훈련 방지:**
1. **5-Fold 교차 검증**: 데이터를 5개로 분할하여 각각 테스트로 사용
2. **Adjusted R-squared**: 변수 개수로 페널티를 적용한 R-제곱
   ```
   Adj R² = 1 - [(1-R²)×(n-1)/(n-p-1)]
   여기서 n=샘플 수, p=변수 수
   ```
3. **조기 종료 조건**: K-Fold 점수가 10번 연속 개선되지 않으면 종료
4. **Tol 허용 오차**: 개선량이 0.000 이하이면 추가 변수 선택 중지
5. **순차적 선택**: 한 번에 하나씩 변수만 추가하여 과적합 방지

2. **계산 효율성**: 이미 필터링된 변수에만 적용

## 3. Spearman 상관관계와 MINE의 필요성 및 비선형 상관관계의 중요성

### 3.1 Spearman 상관관계의 필요성과 한계

**Spearman 상관계수:**
- 순위 기반 상관계수로 비선형 but 단조적 관계 측정 가능
- 이상치에 강한 robustness
- 0~1 사이의 값으로 해석 용이

**한계:**
```python
# 단조적 관계만 감지 가능
y = x²    (x > 0): Spearman = 1 (완벽)
y = sin(x): Spearman ≈ 0 (관계 없다고 판단)
```

### 3.2 MINE (Maximal Information Coefficient)의 필요성

**MINE이 필요한 이유:**
1. **비선형 비단조적 관계 감지**: Spearman이 놓치는 복잡한 관계 포착
2. **광범위한 관계 패턴**: 주기적, 포물선형, 지수형 등 다양한 패턴 감지
3. **정보 이론 기반**: 상호 정보량(Maximal Information) 활용
4. **균등한 해석**: 0~1 사이의 값으로 다양한 관계 비교 가능

**반도체 공정에서의 예시:**
```
온도와 수율의 관계가 최적 온도에서 포물선형을 가질 때:
- Spearman: 낮은 상관계수 (비단조적이므로)
- MINE: 높은 MIC 값 (관계의 존재 감지)
```

### 3.3 비선형 상관관계의 중요성

**반도체 공정에서 비선형성:**
1. **최적 공정 조건**: 온도, 압력 등이 최적값에서 벗어나면 수율 급격히 저하
2. **상호 작용 효과**: 여러 파라미터 간의 복잡한 상호작용
3. **임계점 효과**: 특정 임계값을 넘으면 품질 급격히 변화
4. **포화 현상**: 특정 수준 이상에서는 추가 투입이 효과 없음

**실제 코드에서의 활용:**
```python
# 라인 399-410: MINE 적용

for feature in rf_and_lasso_feature_corr.iloc[:, 1].unique():
    mine.compute_score(final_raw_df[response], final_raw_df[feature])
    mic_result[feature] = mine.mic()  # MIC 값 계산
```

### 3.4 두 방법의 조합 효과

```python
# 라인 411-414: 두 상관계수의 평균 계산
```

- 두 지표의 평균으로 안정적이고 포괄적인 관계 평가

## 4. Lasso가 필요한 이유

### 4.1 특성 선택의 필요성

**Random Forest만으로는 부족한 이유:**
1. **상관관계 문제**: 상관된 변수들이 중요도를 분산하여 과대평가
2. **해의 비일관성**: 실행마다 다른 변수 선택
3. **과적합 위험**: 불필요한 변수 포함 가능성

### 4.2 Lasso의 장점

**L1 규제 (Lasso)의 특징:**
```python
# Lasso 비용 함수
Cost = Σ(y_i - ŷ_i)² + α × Σ|β_j|

# α가 크면: 많은 β_j가 0이 됨 (특성 선택 효과)
# α가 작으면: 더 많은 변수가 선택됨
```

**반도체 공정에서의 효과:**
- **노이즈 변수 제거**: 수율에 영향 없는 공정 파라미터 자동 제외
- **다중공선성 해결**: 상관된 변수 중 하나만 선택
- **단순 모델**: 해석 용이하고 조작 용이
- **일반화 성능**: 테스트 데이터에서 안정적 예측

**Alpha 최적화 (라인 308-316):**

```python
candidate_alphas = np.logspace(-4, 0, 100)  # 0.0001 ~ 1
# 각 alpha에 대해 훈련/테스트 점수 계산
optimal_alpha = lasso_performance.iloc[:, 2].idxmax()  # 최대 테스트 점수
```

**특성 선택 결과 (라인 328-330):**
```python

# 계수가 0이 아닌 변수만 최종 선택
```

### 4.4 Random Forest와 Lasso의 시너지

```
Random Forest (1단계 필터링)
    ↓
수백 개의 변수 → 수십 개의 변수 (중요도 기반)
    ↓
Lasso (2단계 정밀 선택)
    ↓
수십 개의 변수 → 몇 개의 핵심 변수 (계수 기반)
```

**이유:**
- RF: 전체 변수 집합에서 상위 변수 그룹 추출
- Lasso: 추출된 그룹 내에서 실제 수율 영향 변수 정밀 선택

## 5. Step Forward K-Fold가 필요한 이유 및 처리 방식

### 5.1 필요한 이유

**Lasso만으로 부족한 이유:**
1. **순위 효과**: Lasso 계수의 크기가 변수 중요도의 순위를 완벽히 반영하지 않음
2. **상호작용 무시**: 변수 간 상호작용 효과 고려 못함
3. **모델 해석**: 최종 모델의 성능과 안정성이 중요

### 5.2 Step Forward Selection의 작동 방식

**알고리즘 순서:**
```python
# 초기 상태: included = [], curr_best_score = -∞

1단계: excluded = [var1, var2, var3, ...]
    - 각 변수를 개별적으로 테스트
    - 최고 Adj R² 변수 선택: var2

2단계: included = [var2]
    - var2 + 각 변수 조합 테스트
    - 최고 개선량 변수 추가: var5

3단계: included = [var2, var5]
    - var2 + var5 + 각 변수 조합 테스트
    - 개선량 > tol이면 추가, 아니면 종료

... K-Fold 점수가 10번 연속 개선 없으면 종료
```

**코드의 실제 구현 (라인 103-190):**

```python
while True:  # 메인 루프
    # 1. 후보 변수 목록 생성
    excluded = [f for f in X.columns if f not in included]

    # 2. 각 후보 변수 테스트
    for column in excluded:
        train_features = included + [column]  # 새 조합
        model = LinearRegression().fit(X[train_features], y)
        score = model.score(X[train_features], y)
        adj_score = cal_adjusted_r_squared(score, n, p)  # 조정

    # 3. 최고 변수 선택
    best_feature = new_score.idxmax()

    # 4. 개선량 확인
    if (new_score[best_feature] - curr_best_score) > tol:
        included.append(best_feature)
        curr_best_score = new_score[best_feature]
        changed = True

    # 5. K-Fold 교차 검증
    if len(X) >= 30:
        for _, test_idx in kf.split(X, y):
            X_test = X.iloc[test_idx]
            model.fit(X_test[included + [best_feature]], y_test)
            cv_score = cal_adjusted_r_squared(...)

    # 6. 조기 종료 조건
    kf_score = cv_result['Adj_Rsquared'].mean()
    if kf_score > curr_best_kfold_score:
        curr_best_kfold_score = kf_score
    else:
        j += 1
    if j == 10:  # 10번 연속 개선 없으면 종료
        changed = False
```

### 5.3 K-Fold 교차 검증의 역할

**목적:**
1. **과적합 방지**: 훈련 데이터에만 최적화된 변수 선택 방지
2. **일반화 성능**: 미지의 데이터에서도 안정적인 성능 확인
3. **신뢰도 평가**: 5개의 폴드에서 성능의 일관성 확인

**코드에서의 구현 (라인 141-164):**

### 5.4 Step Forward Selection의 효과

**이점:**
1. **순차적 최적화**: 매 단계에서 최적의 변수 추가
2. **해석 용이성**: 변수 추가 순서가 중요도의 순위

## 8. 알고리즘을 순차적으로 처리해야 하는 이유

### 8.1 데이터 의존성

**각 단계의 의존 관계:**
```
RF 중요도 (1단계)
    ↓ 출력: 변수별 중요도 순위
Lasso 선택 (2단계)
    ↓ 입력: RF 상위 변수들
    ↓ 출력: Lasso 선택 변수들
Step Forward (3단계)
    ↓ 입력: Lasso 선택 변수들
    ↓ 출력: 최종 변수 조합
상관계수 분석 (4단계)
    ↓ 입력: Step Forward 결과
```

**이유:**
- 각 단계는 이전 단계의 결과를 입력으로 사용
- 순서를 변경하면 완전히 다른 결과
- 데이터의 차원 축소는 계단식으로 진행

### 8.2 계산 복잡도 관리

**모든 변수에 대해 모든 알고리즘을 적용할 경우:**
```
예시: 1,000개의 변수
- RF: 1,000개 변수 → 계산 가능
- Lasso: 1,000개 변수 → 계산 가능
- Step Forward: 1,000개 변수 →
  최악의 경우 C(1000,1) + C(1000,2) + ... + C(1000,1000) = 2^1000 - 1
  (계산 불가능)
```

**단계적 차원 축소의 효과:**
```
1단계(RF): 1,000개 → 100개 (상위 10%)
2단계(Lasso): 100개 → 20개 (Lasso 계수 비제로)
3단계(Step Forward): 20개 → 5개 (K-Fold 최적화)
```

### 8.3 과적합 방지

**순차적 필터링의 이점:**
1. **노이즈 제거**: 각 단계에서 다른 유형의 노이즈 제거
   - RF: 무관한 변수 제거
   - Lasso: 상관된 변수 중 하나 선택
   - Step Forward: 최적 조합 선택

2. **다양한 관점**: 각 알고리즘의 다른 평가 기준 활용
   - RF: 비선형 중요도
   - Lasso: 선형 계수 크기
   - Step Forward: 선형 조합의 성능

3. **단순 모델**: 단계적 단순화로 해석 용이성 확보

### 8.4 계산 효율성

**병렬적 처리가 어려운 이유:**
- **메모리 제약**: 전체 변수에 대해 모든 알고리즘을 동시 실행 불가
- **시간 복잡도**: 상위 단계의 필터링 없이 하위 단계 실행 불가능
- **최적화**: 단계적 최적화가 전역 최적화보다 계산 효율적

### 8.5 해석 가능성

**순차적 처리의 이점:**
```
최종 모델: Y = 0.5×온도 + 0.3×압력 + 0.2×시간

해석:
1. RF: 온도가 가장 중요 (중요도 0.6)
2. Lasso: 온도, 압력, 시간의 계수가 비제로
3. Step Forward: 3변수 조합이 K-Fold에서 최고 성능

사용자는 각 단계에서 변수가 왜 선택되었는지 이해 가능
```

### 9.1 Adjusted R-squared 계산 오류

### 9.2 Step Forward의 Early Stopping 로직 오류

**문제 (라인 170-178):**

**잘못된 점:**

- 하지만 `changed`가 `True`이면 무조건 다음 루프 진행
- `j`가 10일 때 `changed`를 `False`로 설정하지만, 이는 11번째 루프에서만 적용

**실제 동작:**
```
루프 1: kf_score=0.8 > best=0.7 → best=0.8, j=0
루프 2: kf_score=0.79 > best=0.8? 아니 → j=1

루프 10: kf_score=0.75 > best=0.8? 아니 → j=9
루프 11: kf_score=0.74 > best=0.8? 아니 → j=10, changed=False, 종료

원래 의도: 10번 연속 개선 없으면 종료
실제 동작: 11번째 루프에서 종료 (10번 연속이 아님)
```

**올바른 방법:**
```python

out = Parallel(n_jobs=-1, prefer="threads")(
    delayed(bootstrap_rf)(X_R2SM_si, y_R2SM_si, 0.3) for _ in range(10)
)
result_rfc = pd.concat(out, axis=0, ignore_index=True)
```

**잘못된 점:**
- 각 병렬 작업이 전체 DataFrame을 복사
- 10개의 스레드가 동시에 큰 메모리 사용
- `prefer="threads"`는 GIL(Global Interpreter Lock)로 성능 저하

**올바른 방법:**
```python
# 프로세스 기반 병렬 처리
out = Parallel(n_jobs=-1, prefer="processes")(
    delayed(bootstrap_rf)(X_R2SM_si.values, y_R2SM_si.values, 0.3)
    for _ in range(10)
)
# NumPy 배열로 전달하여 메모리 효율화
```

### **4단계: Step Forward K-Fold 선택**

```python
입력: Lasso 선택된 12개 변수

Step Forward 과정:

Step 1:
- 12개 변수 각각 단독 테스트
- 최고 변수: Annealing_Temp (Adj R² = 0.65)
- K-Fold 검증: 5-fold 평균 Adj R² = 0.62

Step 2:
- Annealing_Temp + 각 변수 조합 테스트
- 최고 추가: Etch_Pressure (Adj R² = 0.74, 개선 +0.09)
- K-Fold 검증: 평균 Adj R² = 0.71

Step 3:
- Annealing_Temp + Etch_Pressure + 각 변수
- 최고 추가: Gas_Flow_AR (Adj R² = 0.79, 개선 +0.05)
- K-Fold 검증: 평균 Adj R² = 0.76

...

Step 6:
- 현재 포함: Annealing_Temp, Etch_Pressure, Gas_Flow_AR,
             Photo_Time, Plasma_Power, Clean_Time
- 다음 변수 추가 시 Adj R² = 0.82 (개선 +0.01)
- K-Fold 검증: 평균 Adj R² = 0.78 (이전과 동일)

Step 7-9:
- K-Fold 점수 개선 없음 (stagnation_count = 3)

Step 10:
- 10번 연속 개선 없음 → 조기 종료

최종 모델: 6개 변수
수율 = 0.45×Annealing_Temp + 0.32×Etch_Pressure + 0.28×Gas_Flow_AR
     + 0.21×Photo_Time + 0.18×Plasma_Power + 0.12×Clean_Time

성능: Adj R² = 0.82, K-Fold Adj R² = 0.78
```
