## 🎮 단계별 재미있는 설명

### 단계 1: 그룹 만들기 (KernelPCA + K-Means) 🎨

**비유:** 500차원 세계에 사는 사람들을 2차원 지도에 옮겨서 이웃끼리 묶기!

```python
# 1단계: 500차원 → 2차원으로 축소
KernelPCA (RBF 커널 사용)
- "온도×압력", "가스×시간" 같은 복잡한 관계까지 포착
- 2차원 평면에 모든 로트를 점으로 찍기

# 2단계: 최적 그룹 수 찾기
Silhouette Score로 테스트:
k=3: 점들이 서로 엉켜있음 (Score 0.45)
k=4: 조금 나음 (Score 0.52)
k=5: 깔끔하게 분리됨! (Score 0.58) ← 최적
k=6: 다시 엉킴 (Score 0.55)

# 3단계: K-Means로 5개 그룹 생성
그룹 0: 저수율 (70-75%) - 2,000개 로트
그룹 1: 중저수율 (75-80%) - 2,500개 로트
그룹 2: 중간수율 (80-85%) - 3,000개 로트
그룹 3: 중고수율 (85-90%) - 1,500개 로트
그룹 4: 고수율 (90-95%) - 1,000개 로트
```

**왜 KernelPCA를 써?**
- 일반 PCA: 선형으로만 줄임 (곧은 줄로만 연결)
- KernelPCA: 비선형으로 줄임 (곡선으로도 연결)
- 예: 온도와 압력의 상호작용도 포착!

**Silhouette Score:**
- -1: 완전히 잘못된 그룹핑
- 0: 그룹끼리 겹침
- 1: 완벽하게 분리됨
- 우리는 0.58로 꽤 좋은 편!

### 단계 3: Logistic Regression 필터링 📊

**비유:** 통계 전문가가 "각 변수가 그룹에 미치는 영향을 수치로 계산!

```python
# 그룹별 계수 해석

변수: 어닐링 온도
그룹0 (저수율): 계수 -2.1 → 온도 낮으면 저수율
그룹1 (중저):   계수 -1.0
그룹2 (중간):   계수  0.0 → 기준
그룹3 (중고):   계수 +1.0
그룹4 (고수율): 계수 +2.1 → 온도 높으면 고수율

해석: "온도가 1표준편차 오르면, 저수율 그룹 확률 ↓, 고수율 그룹 확률 ↑"

변수: 가스 유량
그룹0 (저수율): 계수 +0.3 → 가스 많으면 저수율
그룹4 (고수율): 계수 -0.3 → 가스 적으면 고수율

해석: "가스는 반대로 작동해!"
```

**L2 규제의 역할:**
- 계수가 너무 커지지 않게 제한
- 과적합 방지
- "너무 특정 데이터에만 맞추지 마!"

### 단계 4: Step Forward K-Fold 선택 🚶‍♂️

**비유:** 최고의 그룹 분류기를 만들기 위해서 변수를 하나씩 추가해보기!

```python
현재 후보: 30개 변수
그룹: 5개 (저, 중저, 중간, 중고, 고)

1라운드:
- 어닐링 온도만: 정확도 72%
- 에칭 압력만: 정확도 68%
- 가스 유량만: 정확도 65%
...
🏆 어닐링 온도 먼저 뽑힘!

2라운드:
- 어닐링 온도 + 에칭 압력: 정확도 81% (+9% ↑)
- 어닐링 온도 + 가스 유량: 정확도 76% (+4% ↑)
...
🏆 에칭 압력 추가 뽑힘!

3라운드:
- 온도 + 압력 + 가스 유량: 정확도 86% (+5% ↑)
🏆 가스 유량 추가 뽑힘!

4라운드:
- 온도 + 압력 + 가스 + 플라즈마: 정확도 88% (+2% ↑)
🏆 플라즈마 파워 추가 뽑힘!

5라운드:
- 4개 + 사진 시간: 정확도 89% (+1% ↑)
- Stratified 5-Fold: 평균 83% (변화 없음) 🛑

"그만! 4개가 최적이야!"
```

**Stratified K-Fold:**
```
5개 조로 나눠서 시험 (각 조의 그룹 비율 유지):
조1 (저:중:고 = 2:2:1): 훈련 → 조2로 시험 → 83%
조1: 훈련 → 조3으로 시험 → 84%
...
평균: 83%

이게 진짜 실력이에요! (다른 데이터에서도 잘한다는 뜻)
```

### 단계 5: 상관관계 & MIC 분석 🔗

**비유:** "이 변수가 그 변수랑 얼마나 친해?"를 두 가지 방법으로 확인!

```python
예시: 최종 선택된 4개 변수와 제안된 100개 변수

📊 Spearman 상관계수 (순위 기반):
"어닐링 온도(최종)" ↔ "온도 추세(제안)": 상관계수 0.88
  (매우 강한 양의 상관)

📈 MIC (Maximal Information Coefficient):
"어닐링 온도" ↔ "온도 추세": MIC = 0.72
  (복잡한 관계도 포착)

비교:
- 단순 관계: Spearman이 잘 포착
- 복잡한 관계: MIC가 더 잘 포착
```

**왜 두 가지를 다 쓸까?**
- Spearman: 단순하고 빠른 순위 관계
- MIC: 복잡한 비선형 관계까지 포착
- 두 가지 평균: 더 정확하고 안정적

## 🎭 왜 순차적으로 처리해야 할까?

### 답: 그룹(타겟)이 없으면 분류가 불가능!

```
1단계: 그룹 만들기
    ↓
그룹 레이블 생성: [0, 0, 1, 1, 2, 2, ...]
    ↓ (이걸 가지고 다음 단계)
2단계: RF 분류
    ↓ "어느 그룹이야?"
    ↓
3단계: Logistic Regression
    ↓ "각 그룹의 특성은?"
    ↓
4단계: Step Forward
    ↓ "최적 조합은?"
```

**병렬로 처리하면?**
```
1단계(그룹화) ─┐
               ├─→ 그룹이 없으면 분류 불가!
2단계(RF)      ┤
               │
3단계(LR)      ┤
               └─→ 순서가 반드시 지켜져야 해
```

2. 에칭 압력: 35 mTorr → 50 mTorr (+15 mTorr)

4. 플라즈마 파워: 200W → 250W (+50W)
```

**적용 후 결과:**
```
📈 저수율 그룹 개선 효과:
적용 전: 수율 73% ± 5.2%
적용 후: 수율 86% ± 2.1%
향상: +13%, 편차 -3.1%

💰 경제적 효과:
연간 절감: 약 150억 원
저수율 그룹 → 고수율 그룹 이동: 1,500개 로트
불량률 감소: 60%
```

## 🎓 핵심 개념 요약

### 과적합(Overfitting) 방지 3가지 무기:

```
1️⃣ L1/L2 규제
   L1: 불필요한 변수 계수를 0으로 만들기
   L2: 계수 크기를 제한하기

2️⃣ Stratified K-Fold 교차 검증
   그룹 비율 유지하며 5개 조로 시험

3️⃣ 조기 종료 (Early Stopping)
   10번 연속 개선 없으면 그만
```

### 왜 이렇게 복잡해?

```
🎯 단순 분류만 쓰면:
- 그룹이 없으면 시작 불가
- 변수 수백 개 → 계산 가능하지만 해석 어려움
- 상관관계 → 엉망

🎯 6단계 시스템 쓰면:
- 자동 그룹 발견 (KernelPCA + K-Means)
- 계산 효율적
- 변수 간 상관관계 고려
- 현장에서도 잘 작동!
```

👨‍🏫 5단계: 관계 분석 (상관계수 + MIC)
    ↓

```

## 📚 추가 정보

### Regression vs Clustering

| 특징 | Regression | Clustering |
|------|-----------|-----------|
| 목적 | 수율(연속값) 예측 | 그룹(범주) 분류 |
| 타겟 | 수율 (%) | 그룹 레이블 (0, 1, 2, ...) |
| 방법 | 선형 회귀 | 클러스터링 + 분류 |
| 결과 | "수율 7% 증가" | "저수율→고수율 그룹 이동" |
| 활용 | 공정 최적화 | 그룹별 원인 분석 |

### KernelPCA vs 일반 PCA

```
일반 PCA:
📏 선형으로만 차원 축소
예: 키 ↔ 몸무게 (직선 관계)

KernelPCA (RBF):
🎨 비선형으로도 차원 축소
예: 온도 × 압력 (복잡한 곡선 관계)

반도체 공정은 복잡한 상호작용이 많아서 KernelPCA가 더 적합!
```

### K-Means vs Hierarchical

```
K-Means:
⚡ 빠름 (10,000개 로트에도 OK)
🎯 그룹 수 미리 지정 필요
📊 Silhouette Score로 최적 그룹 수 찾음

Hierarchical:
🐌 느림 (큰 데이터에 부적합)
🌳 덴드로그램 시각화
🔍 계층 구조 파악

반도체 공정은 대량 데이터라 K-Means가 적합!
```
